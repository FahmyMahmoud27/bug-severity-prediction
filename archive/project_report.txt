# Bug Severity Classification - Advanced Project Report

## 1. Project Overview
**What the project does:**
This machine learning project predicts the severity of software bugs based on textual descriptions, titles, and structured categorical attributes. The pipeline ingests raw bug data, preprocesses it, applies natural language processing techniques, and trains multiple classifiers to automatically route and prioritize incoming bugs.

**Problem Statement:**
Bug tracking systems are often inundated with incoming tickets. Manually triaging and assigning a severity level to each bug is time-consuming and subjective. The goal of this project is to build an automated Bug Severity Classification system that standardizes triage, reduces response times for critical vulnerabilities, and ensures engineering efforts are efficiently directed.

---

## 2. Dataset Description
- **Target Variable:** `severity` (Multi-class: Critical, High, Medium, Low).
- **Features Used:**
  - *Textual:* `title`, `description` (combined into `full_text`).
  - *Categorical:* `bug_category`, `tech_stack`, `environment`, `developer_role`, `bug_domain`.
  - *Numeric:* `error_code` (if applicable).

---

## 3. Data Preprocessing Steps
To prepare the dataset for reliable machine learning, the following preprocessing steps were applied:
- **Missing Value Handling:** Missing categorical fields were explicitly filled with `'Unknown'`. Numeric missing values were imputed with `0`. Null textual fields were replaced with empty strings `''`.
- **Text Preprocessing:** All text data (titles and descriptions) were concatenated, converted to lowercase, and stripped of extraneous whitespace to ensure uniform tokenization.
- **Outlier Removal:** An Interquartile Range (IQR) technique was applied to the word count of the combined text to remove extreme outliers (9261 removed), stabilizing the model against anomalies.
- **Clean Pipeline Integration:** A `scikit-learn` `ColumnTransformer` is used to encapsulate `OneHotEncoder(handle_unknown='ignore')` and `TfidfVectorizer` to prevent data leakage and ensure scalable inference processing.

---

## 4. Feature Engineering
- **TF-IDF Vectorization:** The combined `full_text` column was transformed using Term Frequency-Inverse Document Frequency (TF-IDF). Stop words were removed, and the vocabulary was constrained to `max_features=5000` to prevent the curse of dimensionality.
- **Chi-Square (Chi2) Feature Selection:** After transforming categorical and text variables into a sparse matrix, `SelectKBest` with a `chi2` scoring function was utilized to distill the feature space to the top 2000 most statistically relevant features.
